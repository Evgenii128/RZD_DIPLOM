## 📋 О проекте

**РЖД-ТехСервис** — это веб-приложение для автоматизации приёма заявок от клиентов и управления услугами железнодорожной компании. Система состоит из публичной части для подачи заявлений и административной панели для управления.

### ✨ Возможности

#### 🌐 Публичная часть
- Просмотр доступных услуг с ценами
- Подача заявления с выбором нескольких услуг
- Расчёт итоговой стоимости
- Красивая страница подтверждения заявки

#### ⚙️ Административная панель
- 📊 **Дашборд** с KPI-показателями и графиками
- 📝 **Управление заявками** (просмотр, смена статуса, удаление)
- 🛠️ **Управление услугами** (CRUD операции)
- 👥 **Управление клиентами**
- 📋 **Прайс-листы** с гибкой системой позиций
- 📄 **Договоры** с возможностью создания из заявок
- 🧾 **Счета** с отслеживанием оплат

### 🛠️ Технологический стек

| Технология | Назначение |
|------------|------------|
| **Node.js** | Серверная платформа |
| **Express.js** | Веб-фреймворк |
| **MySQL** | База данных |
| **Chart.js** | Графики и диаграммы |
| **Font Awesome** | Иконки |
| **Google Fonts** | Шрифт Inter |
| **bcrypt** | Хеширование паролей |

## 🚀 Быстрый старт

### 📦 Предварительные требования

- Node.js 16 или выше
- MySQL 5.7 или выше
- npm или yarn

### ⚡ Установка

1️⃣ **Клонируйте репозиторий:**

```bash
git clone <url-репозитория>
cd diplom_rzd-main
```

2️⃣ **Установите зависимости:**

```bash
npm install
```

3️⃣ **Настройте переменные окружения:**

Создайте файл `.env` в корне проекта:

```env
PORT=3000
DB_HOST=localhost
DB_NAME=rzd_tehservice
DB_USER=root
DB_PASSWORD=your_password
DB_PORT=3306
ADMIN_USERNAME=admin
ADMIN_PASSWORD=admin123
```

4️⃣ **Запустите приложение:**

```bash
# Обычный запуск
npm start

# Запуск с автоматической перезагрузкой
npm run dev
```

5️⃣ **Откройте в браузере:**

```
🌐 Главная страница:  http://localhost:3000
📝 Подать заявление:  http://localhost:3000/request
⚙️ Панель управления: http://localhost:3000/admin
👤 Личный кабинет:    http://localhost:3000/profile
```

### 🔐 Данные для входа по умолчанию

| Поле   | Значение   |
|--------|------------|
| Логин  | `admin`    |
| Пароль | `admin123` |

## 🔧 API

### 📋 Основные эндпоинты

#### 🔐 Авторизация
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `POST` | `/api/admin/login` | Вход в админ-панель |
| `POST` | `/api/admin/logout` | Выход из админ-панели |
| `GET` | `/api/admin/check-auth` | Проверка авторизации админа |
| `GET` | `/api/admin/current-user` | Текущий админ |
| `POST` | `/api/client/register` | Регистрация клиента |
| `POST` | `/api/client/login` | Вход клиента |
| `POST` | `/api/client/logout` | Выход клиента |
| `GET` | `/api/client/check-auth` | Проверка авторизации клиента |
| `GET` | `/api/client/current-user` | Текущий клиент |

#### 📝 Заявки
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/requests` | Все заявки (с фильтром `?status=`) |
| `GET` | `/api/requests?limit=5` | Последние N заявок |
| `GET` | `/api/requests/:id` | Заявка по ID |
| `POST` | `/api/public/requests` | Подача заявки с сайта |
| `PUT` | `/api/requests/:id/status` | Смена статуса заявки |
| `DELETE` | `/api/requests/:id` | Удаление заявки |

#### 🛠️ Услуги
| Метод | Эндпоинт | Описание |
|-------|----------|----------|
| `GET` | `/api/services` | Все услуги |

| `GET` | `/api/services?search=...` | Поиск по коду/названию |
| `GET` | `/api/services/:id` | Услуга по ID |
| `POST` | `/api/services` | Создать услугу |
| `PUT` | `/api/services/:id` | Обновить услугу |
| `DELETE` | `/api/services/:id` | Удалить услугу |
